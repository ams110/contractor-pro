# 🏭 مصنع صفحات الظهور — SEO + AEO لكبلان

أنت محرّك النمو العضوي لكبلان. شغلتك: تولّد صفحات هبوط محلّية تجرّ المقاول العربي
في إسرائيل عبر جوجل **والذكاء الصناعي** (ChatGPT/Gemini)، بلا أي صرف إعلانات.

## المبدأ الحاسم — جودة لا كمّية

🔴 **جوجل يعاقب «صفحات الباب» (doorway) الرقيقة المكرّرة.** كل صفحة لازم تحمل قيمة
حقيقية متمايزة: مدينة فعلية × تخصّص فعلي + أجور واقعية + أسئلة شائعة حقيقية. **ممنوع
نسخ صفحة وتبديل الاسم فقط.** لو الصفحة ما بتفيد قارئاً بشرياً، لا تولّدها.

🟡 **دفعات صغيرة مقيسة.** لا تطلق مئات الصفحات دفعة وحدة (إشارة سبام لجوجل). دفعة =
~10–25 صفحة، ننشرها، ننتظر إشارة فهرسة/زوّار، بعدها نوسّع. **المنتور هو اللي يقرّر
متى نوسّع** بناءً على الأرقام — مش هذا السكِيل وحده.

## المعمارية (كيف الصفحات مبنية)

كله **مولّد من البيانات** — تزيد بيانات، الصفحات والـSEO والـsitemap تتولّد تلقائياً:

| الملف | الدور |
|------|------|
| `src/lib/cityTradePages.js` | **مصدر الحقيقة**: `CT_CITIES` (مدن + سياق محلّي) و`CT_TRADES` (تخصّصات + أجور). الباقي مولّد: intro/H1/FAQ ثنائية اللغة + `CITY_TRADE_PAGES` + `cityTradeFaqSchema` |
| `src/pages/CityTradeCalculatorPage.jsx` | العرض: ترويسة + حاسبة + قسم أسئلة شائعة مرئي |
| `src/Router.jsx` | يوجّه `/calculator/{city}/{trade}` (مقطعان) لهذه الصفحة |
| `src/lib/seoRoutes.js` | `CITY_TRADE_SEO`: عنوان/وصف/crumb + `schema=FAQPage` لكل صفحة (يقرأه prerender ووقت التشغيل) |
| `scripts/prerender.mjs` | postbuild: HTML ثابت لكل صفحة + حقن JSON-LD + `sitemap.xml` — **تلقائي، لا تلمسه** |

المسار: `/calculator/{citySlug}/{tradeSlug}` (مثال `/calculator/nazareth/balat`).

## كيف تولّد دفعة جديدة

1. **اقرأ الحالي**: `src/lib/cityTradePages.js` — شوف المدن والتخصّصات الموجودة، لا تكرّر.
2. **زِد بيانات متمايزة**:
   - مدينة جديدة في `CT_CITIES`: `{ slug, ar, he, ctx_ar, ctx_he }`. الـ`ctx` = جملة سياق
     محلّي **حقيقية ومختلفة** (ليش هالمدينة فيها مقاولات: شمال/مثلّث/نقب/خبرة معيّنة).
   - تخصّص جديد في `CT_TRADES`: `{ slug, ar, arFull, he, heFull, lo, hi }` — `lo/hi` =
     نطاق أجرة الساعة الواقعي بالشيكل (ابحث/قدّر بصدق، لا ترقام عشوائية).
3. 🔴 **العبري نقيّ 100%**: كل سلسلة عبرية (`he`/`ctx_he`/...) لازم تكون عبرية صرفة —
   ممنوع حرف عربي داخل كلمة عبرية (ي↔י، ر↔ר، و↔ו، ة↔ה...). المصطلحات `מע"מ`/`ביטוח לאומי`
   بحروف عبرية فقط. هاد مبني أصلاً في باني FAQ، بس انتبه للحقول اللي بتكتبها بإيدك.
4. **slugs إنجليزية نظيفة** (URL): حروف صغيرة + شرطات (`umm-al-fahm`, `tel-aviv`).

> 💡 للدفعات الكبيرة بمحتوى أغنى (فقرة محلّية مطوّلة لكل صفحة بدل التركيب الآلي):
> أطلِق وكلاء بالتوازي (Agent tool)، كل وكيل يكتب محتوى مدينة×تخصّص فريد ويرجّعه JSON،
> ثم ادمجه. هيك المحتوى يصير فعلاً متمايزاً مش مركّباً قالبياً. ابدأ بالطريقة الآلية،
> ووسّع للوكلاء لمّا نحتاج عمقاً.

## التحقّق — إجباري قبل النشر

```bash
npm run build      # لازم ينجح: يجمّع + prerender + sitemap
```
- افحص أنّ عدد الصفحات الثابتة وsitemap زادوا بالعدد المتوقّع (السطر `✓ prerender`/`✓ sitemap`).
- افحص صفحة عيّنة: `dist/calculator/<city>/<trade>/index.html` فيها `<title>` صحيح + `FAQPage`.
- فحص نقاء العبري (regex: ما في كلمة تخلط `[֐-׿]` و`[؀-ۿ]`) — انظر أمر node في تاريخ الجلسة.
- تحقّق بصري سريع (preview أو playwright) لصفحة وحدة: الحاسبة + قسم الأسئلة يظهرون بلا خطأ.

## النشر

الصفحات تُنشر بـ**دفع `main`** (Vercel + GitHub Pages ينشران تلقائياً + جوجل يلتقط
الـsitemap الطازج). النشر علني = **خطوة استشارية**: لا تدفع `main` إلا بموافقة المالك
أو ضمن تفويض صريح. بعد الدفع، نبّه المالك بإرسال الـsitemap لـ Google Search Console.

## القياس (الحلقة)

بعد ~أسبوعين من النشر: كم صفحة فُهرست؟ كم زائر عضوي؟ (Search Console + UTM). **لا توسّع
الدفعة الجاية قبل إشارة إيجابية** — صفحات بلا زوّار = توقّف وراجع المحتوى، لا تضاعف.

## السجلّ

بعد أي دفعة: حدّث `docs/GROWTH_MACHINE_PLAN.md` (سطر سجلّ التقدّم) + `docs/founder/shipped.md`
(التاريخ · كم صفحة · الرقم المستهدف=زوّار عضويون · الأساس · الأثر=⏳ تحت المراقبة).
ثم `git add -A && git commit`. لا تدفع `main` بلا إذن.
