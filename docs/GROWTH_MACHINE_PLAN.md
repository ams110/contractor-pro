# 🚀 خطة آلة النمو — Kabblan Growth Machine

> **الوثيقة الحيّة** لآلة النمو الذاتية الدوران لكبلان. تُقرأ وتُحدَّث كل جلسة نمو.
> الهدف: **أول 10 عملاء دافعين** عبر آلة بتدور لحالها بالذكاء الاصطناعي — مش جهد يدوي.

---

## 0. المبدأ (الأساس اللي ما نكسره)

1. **كل ميزة = مدخل مجاني.** ما عندنا أدوات مدفونة ورا قفل — نطلّعها مغناطيس يجرّ المقاول.
2. **ألغِ كل احتكاك.** لا «حمّل تطبيق»، لا «أدخل بياناتك» — قيمة فورية قبل أي طلب.
3. **الذكاء يشغّل، مش اليد.** السكِيلات + العوامل المجدولة (routines) تعمل بالكميّة وبلا تعب.
4. **أرقام حقيقية فقط.** كل قطعة تكتب رقمها بـ Supabase. لا قرار على تخمين.
5. **القمع أولاً.** ما نضخّ صرف مدفوع قبل ما نثبت أنّ أداة → تسجيل تشتغل.

**مقياس النجاح المبكر الأوحد:** أوّل تسجيل حقيقي من أداة مجانية → ثم أوّل عميل دافع.

---

## 1. الآلة = طبقتان

### 🧱 الطبقة 1 — قطع داخل التطبيق (كود React + Vite + Supabase)
أدوات مجانية عامة · onboarding بالصورة · حلقة إحالة. نملكها بالكامل.

### 🤖 الطبقة 2 — الدماغ المؤتمت (كلود كود = محرّك النمو)
- **سكِيلات** = توليد متكرّر (صفحات SEO، رسائل تواصل).
- **Workflows** = دفعات كبيرة بالتوازي (مئات الصفحات بضغطة).
- **Routines (`/schedule`)** = القلب اللي بيدور أسبوعياً بلا طلب (تقرير، قياس، اقتراح).
- **MCP**: Supabase (الأرقام) · Higgsfield (فيديو/صور) · Playwright (تحقّق) · بحث ويب.

**العمود الفقري:** Supabase = مصدر كل رقم حقيقي. التقرير المجدوَل بيجمّعهم.

## 2. خارطة المراحل

### Phase 1 — أوّل أداة مجانية (الأساس) 🟢 نبدأ هون
- **نبني:** صفحة عامة `/tools/salary` (حاسبة راتب مقاول) + `/tools/vat` (حاسبة מע"מ) — بلا تسجيل، تعيد استعمال `SalaryCalculator.jsx` + محرّك الضريبة (`helpers.js`). CTA → `/register?utm=tool-salary`. SEO عبر `seoRoutes.js` (موجود) + sitemap تلقائي (prerender).
- **نستعمل:** جلسة كلود كود (بناء) + Playwright (تحقّق).
- **يشتغل في:** التطبيق (Vercel).
- **مقياس النجاح:** زوّار `/tools` · نسبة الضغط على CTA · تسجيلات بـ utm=tool.
- **الوقت:** جلسة–جلستين.

### Phase 2 — مصنع صفحات SEO (سكِيل)
- **نبني:** سكِيل `seo-page-factory` يولّد صفحات هبوط لكل (تخصّص × مدينة) تشير للأداة، بـ SEO محلّي.
- **نستعمل:** سكِيل + Workflow (توليد متوازٍ) + Higgsfield (صور اختياري).
- **مقياس:** صفحات مفهرسة · ظهور عضوي (Google Search Console) · زوّار عضويين.
- **الوقت:** بناء السكِيل ~جلسة، بعدها بضغطة.

### Phase 3 — Routine تقرير النمو (القياس) ⏰
- **نبني:** routine أسبوعي (`/schedule`) يسحب من Supabase: زوّار الأدوات + تسجيلات لكل مصدر + شو اشتغل → تقرير 5 أسطر + يقترح صفحة جديدة.
- **نستعمل:** Routine (سحابي) + Supabase.
- **⚠️ تحقّق:** نتأكّد إنه فعلاً بيوصل للأرقام (مفتاح خدمة مضبوط أو سكربت مرفوع) — مش نفترض.
- **مقياس:** التقرير نفسه = العمود الفقري للقياس.
- **الوقت:** جلسة + تحقّق.

### Phase 4 — onboarding بالصورة (يفكّ أكبر احتكاك)
- **نبني:** نكبّر edge `scan-receipt` (OCR موجود) → تدفّق «صوّر فواتيرك/دفترك» يبني الحساب بدقايق.
- **مقياس:** تحويل تسجيل → حساب مُفعَّل.
- **الوقت:** 2–3 جلسات.

### Phase 5 — حلقة الإحالة (نمو مجاني مدمج)
- **نبني:** جدول `referrals` + بوّابة العامل «أنا مقاول، جرّب مجاناً» + مكافأة (شهر مجاني) + تتبّع.
- **مقياس:** معامل الانتشار (دعوات لكل مستخدم · تحويلها).
- **الوقت:** ~جلستين.

### Phase 6 — التواصل الآلي من سجلّ المقاولين ⚖️ (مشروط قانونياً)
- **نبني:** سحب بيانات **רשם הקבלנים** → تخصيب → رسائل مشخصنة بالذكاء → إرسال عبر **Resend** (مدمج) أو WhatsApp/SMS (Twilio).
- **نستعمل:** `deep-research` (فحص قانوني أول) → سكِيل (توليد رسائل) → Resend/Twilio.
- **🔴 ضابط قانوني حاسم:** قانون الـ**ספאם** الإسرائيلي بيتطلّب غالباً موافقة مسبقة (opt-in) للتواصل التجاري. هاي المرحلة **مقفولة** لحدّ ما نتحقّق قانونياً من: (أ) شرعية استعمال بيانات السجلّ، (ب) متطلّبات الـopt-in. ما نطلقها قبل هيك.
- **الوقت:** بعد الفحص القانوني.

### Phase 7 — AEO + فيديو بالآلاف (متقدّم/مستمر)
- **نبني:** تحسين المحتوى عشان مساعدات الذكاء (ChatGPT/Gemini) ترشّح كبلان · توليد نسخ فيديو بالآلاف عبر Higgsfield واختبارها برمجياً.
- **مقياس:** ذكر كبلان كجواب · أداء نسخ الفيديو.
- **الوقت:** مستمر.

---

## 3. العمود الفقري للقياس (Supabase)
- كل أداة/صفحة تكتب: مصدر الزائر (UTM) + الحدث (استعمل أداة / ضغط CTA / سجّل).
- لوحة `/admin` + RPC `admin_get_stats` = مصدر التسجيلات الحقيقية لكل مصدر.
- routine النمو (Phase 3) يجمّعهم أسبوعياً.

---

## 4. المخاطر والضوابط
| الخطر | الضابط |
|------|--------|
| ضخّ صرف على قمع غير مثبت | فيتو القمع: لا مدفوع قبل ما أداة → تسجيل تشتغل |
| قانون الـספאם الإسرائيلي (تواصل بارد) | Phase 6 مقفولة لحدّ فحص قانوني (opt-in) |
| شرعية سحب بيانات السجلّ | `deep-research` يتحقّق قبل أي سحب |
| كسر الإنتاج | كل بناء على برانش + تحقّق Playwright + `npm run build` قبل الدفع |
| قرارات على أرقام وهمية | كل مرحلة تكتب رقمها الحقيقي بـ Supabase |

## 5. شو محتاج منك (المؤسس)
1. **الآن:** ضوء أخضر لبناء Phase 1 (على برانش، ما ألمس الإنتاج لحدّ ما تشوفها).
2. **لاحقاً (Phase 3):** وصول لمفتاح Supabase للـrouting (نضبطه سوا).
3. **لاحقاً (Phase 6):** قرار قناة التواصل + راحتك القانونية بعد الفحص.
