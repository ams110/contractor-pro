# اختبار الواجهة — Playwright

في طريقتين لاختبار التطبيق على المتصفح، وكلاهما مركّب جاهز:

1. **Playwright MCP** — كلود كود يفتح متصفح حقيقي ويجرّب التطبيق زي مستخدم بشري (تفاعلي).
2. **اختبارات E2E جاهزة** — ملفات اختبار تشتغل تلقائياً بأمر واحد وتفحص الفلوز الأساسية.

---

## 1) Playwright MCP — كلود يجرّب التطبيق بنفسه

مركّب في `.mcp.json` تحت اسم `playwright`. لما تشغّل **Claude Code على جهازك**، بيقدر يفتح
متصفح حقيقي، يدخل على `localhost`، يضغط أزرار، يعبّي فورمات، ويتنقّل بين الصفحات — وبعدها
يعطيك تقرير بالأخطاء اللي لقاها.

### الاستخدام

```bash
# 1. شغّل التطبيق
npm run dev          # → http://localhost:3000

# 2. افتح Claude Code في نفس المجلد، ووافق على تشغيل سيرفر playwright لما يطلب،
#    وبعدها اطلب منه مثلاً:
```

> «افتح http://localhost:3000، جرّب تسجيل الدخول وإنشاء حساب جديد، وتنقّل بين
> الصفحات، وقلّي إيش في أخطاء أو مشاكل بالتصميم.»

> ملاحظة: أول مرة، `npx @playwright/mcp@latest` بينزّل السيرفر تلقائياً. لو المتصفح
> مش منزّل: `npx playwright install chromium`.

## 2) اختبارات E2E الجاهزة

الملفات في `tests/e2e/`. بتشغّل Vite dev server تلقائياً (port 3000) قبل ما تبدأ،
وبتشتغل على viewport موبايل + ديسكتوب.

### الأوامر

```bash
npm run test:e2e          # شغّل كل الاختبارات (headless)
npm run test:e2e:ui       # واجهة تفاعلية لمتابعة الاختبارات
npm run test:e2e:report   # افتح آخر تقرير HTML
```

تشغيل ملف أو متصفح محدد:

```bash
npx playwright test tests/e2e/navigation.spec.js
npx playwright test --project=mobile-chrome
npx playwright test --headed         # شوف المتصفح وهو يشتغل
```

### شو بتغطّي

| الملف | التغطية |
|------|---------|
| `landing.spec.js`   | صفحة الهبوط: العنوان، الأزرار، شريط الإحصائيات، scroll للمميزات |
| `navigation.spec.js`| التنقّل: دخول / تسجيل / أسعار + فتح المسارات مباشرة |
| `auth-forms.spec.js`| فورمات الدخول والتسجيل: الحقول، إظهار الباسورد، استرداد، عضو فريق، تبديل اللغة، تحقّق طول كلمة المرور |

> الاختبارات client-side بس (تنقّل + تحقّق) — ما بتلمس الباكند، فبتشتغل بدون
> إعداد Supabase. لإضافة فلوز تلمس الباكند (دخول فعلي، إضافة عامل، إلخ) لازم
> بيئة اختبار/مستخدم تجريبي.

### الإعداد على سيرفر خارجي

لو بدك تشغّل الاختبارات ضد رابط منشور بدل الـ dev server المحلي:

```bash
E2E_BASE_URL=https://your-deployment.vercel.app npm run test:e2e
```
