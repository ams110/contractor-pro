# تقرير تدقيق SEO — Contractor Pro

> مرجع: [دليل جوجل لبدء تحسين محركات البحث](https://developers.google.com/search/docs/fundamentals/seo-starter-guide).
> النطاق: `https://app.linko.services`. آخر تحديث: 2026-06-19.

تطبيق Contractor Pro هو SPA (React + Vite) مع **prerender ثابت** لكل مسار تسويقي وقت
البناء، فتظهر ميتا صحيحة للزواحف وروابط المشاركة. هذا التقرير يطابق كل بند من دليل
جوجل بحالة التطبيق، ويفصل المتبقّي **التشغيلي** (خارج الكود).

---

## 1) مساعدة جوجل على إيجاد المحتوى

| البند | الحالة | التفاصيل |
|------|:-----:|---------|
| الزحف والفهرسة | ✅ | SPA يخدم `index.html`، وجوجل يرندر JS. + prerender ثابت لـ9 مسارات للزواحف غير الرندرة (واتساب/فيسبوك/تويتر). |
| sitemap.xml | ✅ | **مولّد تلقائياً** وقت البناء من `seoRoutes.js` (مصدر واحد)، بتاريخ `lastmod` طازج. معلَن في `robots.txt`. |
| robots.txt | ✅ | `Allow: /` + حجب الأدوات الداخلية (`/admin`, `/adstudio`, `/adreel`, `/demoshot`, `/welcome`, `?portal`, `?worker`). |
| الموارد (CSS/JS) غير محجوبة | ✅ | لا حجب لأي أصول؛ جوجل يرى الصفحة كالمستخدم. |
| تحقّق الملكية | ✅ | `google-site-verification` meta + ملف `google2e9ae507788087c4.html`. |

## 2) تنظيم الموقع

| البند | الحالة | التفاصيل |
|------|:-----:|---------|
| روابط وصفية | ✅ | `/pricing`, `/blog`, `/terms`, `/privacy`, `/refund`, `/contact` — كلمات مفهومة لا معرّفات عشوائية. |
| تجميع منطقي | ✅ | بنية مسطّحة مناسبة لموقع صغير (<10 صفحات عامة). |
| تكرار المحتوى | ✅ | `<link rel="canonical">` مضبوط لكل مسار عبر prerender. لا نسخ مكرّرة. |

## 3) محتوى مفيد ومثير

| البند | الحالة | التفاصيل |
|------|:-----:|---------|
| نصّ منظّم وسهل | ✅ | صفحة هبوط بعناوين وأقسام واضحة، RTL عربي. |
| محتوى فريد | ✅ | محتوى أصلي مخصّص للمقاول العربي في إسرائيل. |
| توقّع كلمات البحث | ✅ | مصطلحات مزدوجة (مע"מ/ضريبة القيمة المضافة، עוסק פטור/عوسك باتور) في الميتا والمحتوى. |
| إعلانات غير مزعجة | ✅ | لا إعلانات على الموقع. |
| روابط داخلية | ✅ | تنقّل بين الهبوط/الأسعار/المدوّنة/القانونية بنصّ رابط وصفي. |
| 🟡 عمق المدوّنة | تشغيلي | `/blog` صفحة واحدة حالياً. كل مقال جديد = فرصة كلمات بحث جديدة (انظر التوصيات). |

## 4) المظهر في نتائج البحث

| البند | الحالة | التفاصيل |
|------|:-----:|---------|
| `<title>` فريد لكل صفحة | ✅ | من `ROUTE_SEO` عبر prerender ووقت التشغيل (`useSeo`). |
| meta description | ✅ | فريدة وموجزة لكل مسار. |
| Open Graph / Twitter | ✅ | `og:*` + `twitter:card=summary_large_image` لكل مسار. |
| صورة مشاركة | ✅ | `og-image.png` 1200×630 + `og:image:alt`. |

## 5) الصور والوسائط

| البند | الحالة | التفاصيل |
|------|:-----:|---------|
| صور قرب نصّ ذي صلة | ✅ | الواجهة قائمة على أيقونات Lucide متجهية (لا صور خام تحتاج فهرسة). |
| alt وصفي | ✅ | `og:image:alt` موجود؛ لا `<img>` محتوى بلا alt على الصفحات العامة. |
| الفيديو | لا ينطبق | لا صفحات فيديو مستقلة. |

## 6) البيانات المنظّمة (Structured Data) — تجاوز الأساسيات

| النوع | الحالة | المصدر |
|------|:-----:|--------|
| Organization | ✅ | `index.html` (`@graph`) |
| WebSite | ✅ | `index.html` |
| SoftwareApplication + AggregateOffer | ✅ | `index.html` (أسعار 129–499 ILS) |
| FAQPage | ✅ | مبني من `FAQ_ITEMS` (`seoRoutes.js`) على الهبوط |
| BreadcrumbList | ✅ | `breadcrumbFor()` تلقائياً لكل مسار فرعي |

> هذه تؤهّل الصفحات لميزات بحث محسّنة (نتائج غنية). تحقّق منها بـ
> [Rich Results Test](https://search.google.com/test/rich-results).

## ما لا يجب التركيز عليه (حسب جوجل) — وحالتنا متوافقة

- **meta keywords**: موجودة في `index.html` لكن جوجل يتجاهلها — غير ضارّة (لا keyword stuffing).
- **TLD/كلمات في الدومين**: `app.linko.services` مناسب للعلامة — لا قلق.
- **طول المحتوى / عدد العناوين / ترتيبها / E-E-A-T كعامل ترتيب**: لا تحسين قسري مطلوب.

## الخلاصة: الكود مغطّى ✅

كل البنود **القابلة للتنفيذ بالكود** في دليل جوجل مطبّقة. التحسين الوحيد المنفّذ في
هذه الدفعة: **توليد `sitemap.xml` تلقائياً** بتاريخ طازج (كان ثابتاً يدوياً).

## المتبقّي = تشغيلي (خارج الكود) — مرتّب بالأولوية

1. **🔴 Google Search Console** (الأهم): فعّل الحساب، قدّم `sitemap.xml`، راقب الفهرسة
   والأداء وأخطاء الزحف. خطوات مفصّلة في رسالة المحادثة / [دليل البدء](https://developers.google.com/search/docs/monitor-debug/search-console-start).
2. **🟠 محتوى المدوّنة**: انشر مقالات تستهدف نوايا بحث المقاولين (مثلاً «كيف أحسب راتب
   عامل مع الإضافي»، «מע"מ للعוسك مורشيه شرح مبسّط»، «حدّ עוסק פטור 2026»). كل مقال =
   كلمات بحث جديدة وصفحة قابلة للفهرسة. أضف مساراً لكل مقال في `ROUTE_SEO` فيدخل الخريطة تلقائياً.
3. **🟠 روابط خارجية (Backlinks)**: مذكور في الدليل كأقوى إشارة اكتشاف. مجموعات مقاولين،
   دلائل أعمال محلية، شراكات.
4. **🟡 الترويج**: السوشال ميديا، الكلمة المنطوقة، URL على بطاقات العمل/المواد المطبوعة.
5. **🟡 مراقبة دورية**: راجع تقارير Search Console كل أسبوعين، حدّث المحتوى القديم.

> ملاحظة: عند إضافة أي مسار عام جديد في `ROUTE_SEO` (بلا `noindex`)، يدخل الـsitemap
> والـprerender **تلقائياً** — لا صيانة يدوية للخريطة.
