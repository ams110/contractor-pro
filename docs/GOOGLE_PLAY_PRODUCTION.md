### البوّابة 2 — سياسة الدفع (Google Play Billing) — خطر الرفض الأول

التطبيق بيبيع اشتراكات عبر **Paddle** (مش نظام دفع جوجل). جوجل عندها سياسة صارمة:
أي تطبيق بيبيع محتوى رقمي/اشتراك **من جوّا التطبيق** لازم يستعمل **Google Play Billing**.

- ✅ **الجانب المطمئن:** كبلان أداة **عمل/SaaS** بتُستهلك على الويب + التلفون، والاشتراك خدمة
  مستمرّة عبر المنصّات — هاي الفئة عادةً بتمرّ. وكمان التطبيق **عبر مراجعة الاختبار المغلق أصلاً**.
- 🔴 **الخطر:** مراجعة الإنتاج أصرم. لو جوجل شافت زر «ترقية/اشترك» بيفتح دفع Paddle **جوّا** التطبيق،
  ممكن ترفض.
- **الخيار الأكثر أماناً (موصى به مبدئياً):** التجربة المجانية 14 يوم تخلّي التطبيق قابل للاستعمال
  بالكامل بلا أي دفع داخلي — فبنطلق للإنتاج، ولو جا اعتراض من جوجل بنتعامل معه ساعتها (إمّا إخفاء
  مسار الدفع داخل نسخة الأندرويد، أو دمج Google Play Billing لاحقاً).

> 📌 قرار: **بنكمّل للإنتاج.** بس كن جاهز لردّ محتمل من جوجل بخصوص الدفع. (كلود بيساعدك بالردّ/التعديل.)

## 📋 الخطوات بالترتيب (داخل Play Console)

### الخطوة 0 — هل تحتاج نسخة (AAB) جديدة؟

**على الأغلب لأ.** أسهل طريق للإنتاج = **ترقية (Promote)** نسخة الاختبار المغلق الموجودة → Production.
ما بدّك تبني AAB جديد إلا إذا:
- اسم/أيقونة التطبيق على الجهاز لسّا **"Contractor Pro"** (يعني نسخة الاختبار اتبنت قبل تغيير الاسم لـ«كبلان») → لازم نبني نسخة جديدة.
- بدك ترفع `versionCode` لأي سبب.

> لو احتجنا نسخة جديدة: تبويب **Actions** بـ GitHub → **Build Android (TWA)** → **Run workflow** →
> حُطّ `versionName` أكبر (مثلاً `1.0.2`) واترك `versionCode` فاضي (بيتولّد تلقائياً). نزّل الـ `.aab` من الـ Artifacts.

### الخطوة 1 — أكمل كل "محتوى التطبيق" (App content)

Play Console → **Policy and programs → App content**. لازم كل بند يكون ✅ أخضر:

| البند | الجواب الجاهز |
|------|----------------|
| **Privacy policy** | `https://app.linko.services/privacy` |
| **Ads** | التطبيق **لا يحتوي إعلانات** → "No, my app does not contain ads" |
| **App access** | التطبيق محمي بتسجيل دخول → وفّر **حساب تجريبي** لمراجعي جوجل (إيميل + كلمة سرّ لحساب فيه بيانات ديمو). **مهم جداً** وإلا بيرفضوا. |
| **Content ratings** | عبّي الاستبيان (تفاصيله تحت) → النتيجة المتوقّعة: **للجميع / Everyone 3+** |
| **Target audience** | الفئة العمرية: **18 وأكبر** (أداة عمل، مش للأطفال) |
| **Data safety** | عبّي النموذج (تفاصيله تحت) |
| **Government apps** | لأ |
| **Financial features** | التطبيق **يدير** بيانات مالية للعمل لكنه **ليس** تطبيق دفع/قروض/استثمار → غالباً "None of these". (لو سأل عن إدارة ميزانية شخصية، هو أداة محاسبة أعمال — اختر الأقرب أو None.) |
| **Health** | لأ |

### الخطوة 2 — أكمل تفاصيل المتجر (Store listing)

Play Console → **Grow → Store presence → Main store listing**. (النصوص الجاهزة بالأسفل §وصف المتجر.)
- **App name** (اسم التطبيق) ← انسخ من تحت
- **Short description** (الوصف القصير) ← انسخ
- **Full description** (الوصف الكامل) ← انسخ
- **App icon** (512×512) — موجود: `public/icon-512`/`pwa-512.png`
- **Feature graphic** (1024×500) — **مطلوب وإلزامي** (كلود بيولّده — اسأله)
- **Phone screenshots** (2–8 صور، الأقل صورتين) — كلود بيولّدها عبر سكِيل `promo-shots`

### الخطوة 3 — أنشئ إصدار الإنتاج (Production release)

Play Console → **Release → Production → Create new release**:
- **الطريق الأسهل:** اضغط **"Add from library"** واختر نفس الـ AAB من الاختبار المغلق (ترقية).
  - أو ارفع الـ AAB الجديد لو بنيت واحد بالخطوة 0.
- **Release name**: مثلاً `1.0.2`
- **Release notes** (ملاحظات الإصدار) ← انسخ من §ملاحظات الإصدار تحت
- اضغط **Next** → **Save** → **Review release**.

### الخطوة 4 — قدّم للمراجعة (Submit)

- إذا حسابك **فردي** وما كمّلت الـ12×14: بيظهرلك **"Apply for production access"** — عبّي الاستمارة
  (بيسألوا عن خبرة الاختبار: كم مختبِر، شو الفيدباك). جاوب بصدق.
- إذا **مؤسّسة** أو خلّصت المتطلّب: اضغط **Send for review**.
- المراجعة بتاخد عادةً **بضع ساعات حتى 7 أيام**.

## 🔐 نموذج أمان البيانات (Data Safety) — جاهز للنقل

في Play Console → **App content → Data safety**. جاوب: **نعم، التطبيق يجمع/يستخدم بيانات**.

**إعدادات عامّة:**
- البيانات مشفّرة أثناء النقل (Encrypted in transit): **نعم**
- يوفّر طريقة لطلب حذف البيانات (Data deletion): **نعم** → الرابط: `https://app.linko.services/delete-account`

**أنواع البيانات المجموعة** (كلها: تُجمع = نعم · تُشارَك = لا · غرضها = تشغيل التطبيق/إدارة الحساب):

| النوع (Category → Type) | تُجمع | تُشارَك | إلزامي؟ | الغرض |
|---|---|---|---|---|
| Personal info → **Name** | نعم | لا | إلزامي | تشغيل التطبيق + إدارة الحساب |
| Personal info → **Email address** | نعم | لا | إلزامي | تشغيل التطبيق + إدارة الحساب + المصادقة |
| Personal info → **Phone number** | نعم | لا | اختياري | تشغيل التطبيق (تواصل واتساب مع العمّال) |
| Financial info → **Other financial info** | نعم | لا | إلزامي | تشغيل التطبيق (رواتب/مصاريف/مقبوضات العمل) |
| App activity → **App interactions** | نعم | لا | اختياري | التحليلات / تحسين الأداء |
| Device or other IDs → **Device or other IDs** | نعم | لا | اختياري | التحليلات (Google Analytics — بموافقة) |
