## 🗂️ تفاصيل / الخطوة الجاية

### ✅ تحديث 25/06/2026 — القمع انصلح (P0) + الإسناد اكتمل (P2)
- **P0 — حائط التسجيل اتشال** (PR #229): ديمو عام بلا تسجيل (`/demo`) + تسجيل فوري بلا تأكيد إيميل + Google OAuth + فورم مخفّف. → الزائر صار يقدر يشوف التطبيق ويسجّل بضغطة بدل ما يهرب من الحائط.
- **P2 — «نوقف العمى» اكتمل** (هاد التغيير): طبقة إسناد `src/lib/attribution.js` بتلتقط `utm_*`/`ttclid`/`gclid` بأول زيارة (first-touch، localStorage) وبتُرفقها بـ: (1) `user_metadata` لكل مسجّل جديد (تظهر بـ`auth.users`/لوحة `/admin`) + (2) أحداث `sign_up`/`generate_lead` بـGA4. → **صار نقدر نعرف أي إعلان/حملة جاب كل تسجيل** — هو الرقم الذهبي الناقص.
- **الناقص قبل ما نرجع نموّل**: (أ) التحقّق إنّ «تأكيد الإيميل» مطفأ فعلاً بلوحة Supabase Auth (الكود جاهز بس الإعداد بالداشبورد). (ب) اختبار حيّ: افتح رابط بـ`?utm_source=tiktok&utm_campaign=test` → سجّل → تأكّد إنّ `attr_source=tiktok` ظهر بالـuser_metadata.
- **عند أول إعلان جديد**: ضع UTM على رابط الإعلان (`?utm_source=tiktok&utm_medium=paid&utm_campaign=<اسم الزاوية>&utm_content=<نسخة الإبداع>`) → إبداع شاشة-أولاً نظيف (بلا علامة منافِسة) → سجّل النتيجة + **CPL الحقيقي من التسجيلات الموسومة** هون.

> القرار: 🔧 القمع + القياس صاروا جاهزين. الخطوة المنطقية الجاية = **إنتاج إبداع جديد + إطلاق صغير موسوم بـUTM** ونقيس CPL الحقيقي لأول مرّة.
