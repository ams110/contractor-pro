# 🎯 خطة تحويل التطبيق — من «مشاهدة» إلى «مشترك دافع»

> **وثيقة تنفيذية** لتشات تطوير منفصل. الهدف: إصلاح القمع المكسور الي اكتشفناه بالتحليل،
> بحيث زائر تيك توك يتحوّل: **يشوف → يسجّل → يدخل → يوصل لحظة القيمة → يشترك بالخطة المدفوعة.**
>
> 🔴 **خلفية حاسمة (من تحليل 14–16/06/2026)**: حملة تيك توك جابت **41K مشاهدة + 362 زيارة صفحة** بـ₪136
> — و**صفر تسجيل** (مؤكّد من `auth.users`: لا تسجيل واحد منذ 11/06). المشكلة **ليست بالإعلانات** (رخيصة وفعّالة)
> بل **بالقمع**: ولا زائر حتى بدأ يعبّي فورم التسجيل. التمويل موقوف حتى يُصلَح القمع.

لازم يفتح حساب كامل (اسم+إيميل+باسوورد+**تأكيد إيميل**) عشان **بس يشوف** التطبيق → بيهرب. هو بدّو يشوف، مش يلتزم.

## 2. الرحلة المستهدفة (Funnel) ومقاييس كل مرحلة

```
فيديو (هوك شاشة)  →  ديمو بدون حساب  →  تسجيل بضغطة  →  Onboarding للحظة «آها»  →  تجربة 14 يوم  →  ترقية مدفوعة
   CTR ~1%            % دخلوا الديمو      % سجّلوا         % وصلوا أول قيمة            % فعّالين أسبوع     % تحويل لمدفوع
```

كل مرحلة لازم يكون إلها **حدث محسوب** (analytics event) — حالياً إحنا **عميان** عن كل هالقمع.

## 3. خطة التنفيذ (مرتّبة بالأثر/الجهد)

### 🥇 P0 — مكاسب سريعة (تحلّ جذر «صفر تسجيل») — ابدأ هون

1. **«جرّب بدون تسجيل» — ديمو عام تفاعلي** ⭐ الأهم
   - عندنا أصلاً مسار `/demoshot` ببيانات وهمية متماسكة (انظر `scripts/demo-shots.mjs` و`CLAUDE.md`).
   - المطلوب: مسار عام مثل `/demo` أو `?demo` يفتح **التطبيق الحقيقي بوضع قراءة ببيانات وهمية** بدون auth — مع شريط علوي ثابت «هاي نسخة تجريبية · **سجّل مجاناً لتبدأ بمصلحتك**» (CTA دائم).
   - زر «ابدأ مجاناً» بصفحة الهبوط ممكن يصير → الديمو أولاً، والـCTA داخل الديمو → التسجيل.
   - **الأثر**: بيحوّل الفضول لتجربة فورية بدل حائط التسجيل.

2. **شيل حاجز تأكيد الإيميل**
   - Supabase → Authentication → Providers/Email → **أطفِ "Confirm email"** (دخول فوري؛ التأكيد لاحقاً اختياري).
   - عدّل `LoginScreen.jsx` (سطر ~165–167): بدل رسالة «تحقّق من بريدك للتفعيل» → **دخول مباشر** بعد `signUp` (استدعِ `signIn` تلقائياً أو وجّه لـ`/app`).
   - ملف متعلّق: `src/hooks/useAuth.js` (`signUp`) · `src/screens/auth/LoginScreen.jsx`.

3. **خفّف فورم التسجيل**
   - إيميل + باسوورد فقط (اجمع الاسم/التفاصيل داخل onboarding لاحقاً).
   - **Google OAuth بضغطة** (`supabase.auth.signInWithOAuth({ provider:'google' })`) — أقوى مزيل احتكاك على الموبايل.

4. **صفحة هبوط مخصّصة لإعلانات تيك توك**
   - مسار خفيف موبايل-أول (مثلاً `/start` أو `/l/tiktok`) — هوك واحد + GIF/فيديو شاشة + زر «شوف التطبيق» (ديمو) + CTA تسجيل. بلا قوائم/تشتيت.
   - وجّه رابط البايو/الـPromote لهون (مع UTM — انظر P2).

### 🥈 P1 — التفعيل (Activation): يوصلهم للحظة «آها» بسرعة

5. **Onboarding موجّه بعد التسجيل** — بدل شاشة فاضية:
   - خيار «جهّز لي بيانات تجريبية أبدأ منها» (يعبّي مشروع+عمّال وهميين قابلين للحذف) → يشوف القيمة فوراً.
   - أو معالج 3 خطوات: أضف مشروع → أضف عامل → شوف الراتب/الضريبة تتحسب. **اللحظة «آها» = يشوف رقم محسوب تلقائياً.**
   - `FirstTimeSetup.jsx` نقطة الدخول الطبيعية للتوسعة.

6. **بوّابة العامل كطُعم انتشار** — بعد ما يضيف عامل، حثّه يبعتله رابط البوّابة (واتساب) — `whatsapp.js` جاهز. كل عامل = نقطة لمس جديدة.

### 🥉 P2 — القياس والإسناد (Attribution): نوقف العمى

7. **UTM على كل رابط إعلان** (`?utm_source=tiktok&utm_campaign=...`) — خزّنها بأول زيارة (localStorage) واكتبها مع التسجيل.
8. **أحداث تحويل محسوبة**: زيارة هبوط · دخول ديمو · بدء تسجيل · إتمام تسجيل · أول إجراء · بدء تجربة · ترقية. اربطها بـ`analytics.js` (GA4 موجود) — وهيك **نشوف** القمع بدل التخمين.
9. **TikTok Pixel/Events API** (لاحقاً) لتحسين الإعلان على «التسجيل» لا «النقرة».

### 🏆 P3 — التحويل للمدفوع (Monetization)

10. **نخبة المنتج موجودة**: `usePlanStore` (تقييد ميزات) · تجربة 14 يوم (`useOrganization`) · `FeatureGate`/`PortalUpsell` · `PricingPage` + Paddle. المطلوب **تفعيل اللحظات**:
    - **حثّ ترقية عند لحظة القيمة** (مثلاً عند محاولة إضافة العامل رقم 2 على التجربة، أو فتح ميزة Pro) — مش إزعاج عشوائي.
    - **تذكير قبل انتهاء التجربة** (إشعار داخل + واتساب) — `useNotifications`/`usePushNotifications`.
    - **وضوح التسعير**: «أرخص من نص محاسب» على صفحة الترقية (من `MARKET_RESEARCH.md`).
    - **حلقة الإحالة**: حافز ثنائي «ادعُ مقاول، الاثنين شهر مجاني» — أعلى K-factor (انظر `docs/ADVERTISING_STRATEGY.md §3.5`).
