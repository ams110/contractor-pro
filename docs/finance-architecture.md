## 🎯 الرؤية النهائية

**ما هو هذا القسم:**
نظام إدارة مالية عملي لمقاول يدير أكثر من مصلحة —
يسجّل مدخولاته ومخروجاته، يأرشف فواتيره الواردة للمحاسب،
ويولّد قسائم رواتب لعماله.

**ما ليس هذا القسم:**
- ❌ لا يولّد فواتير ضريبية — عندك تطبيق منفصل لذلك
- ❌ ليس نظام محاسبة معقد

**قواعد الربط:**
- العملاء والمشاريع مشتركين — الربط بهم اختياري في أي سجل
- كل سجل مالي مرتبط بمصلحة محددة

## المستوى الأول: قاعدة البيانات

### ❶ المصالح / الشركات — الجذر

```
businesses
├── id, user_id
├── name           — اسم المصلحة
├── business_type  — osek_patur / osek_moreh / hevra
├── reg_number     — رقم ע.מ / ח.פ
├── phone, address
├── logo_url
├── bank_name, bank_account, bank_branch
└── created_at
```

---

### ❷ المدخولات — ما دخل

```
income_entries
├── id, business_id
├── amount, date
├── source         — client / project_payment / advance / other
├── method         — cash / transfer / check / app
├── client_id      — (اختياري) ← clients
├── project_id     — (اختياري) ← projects
├── note
├── proof_url      — صورة إثبات الدفع
└── created_at
```

---

### ❸ المخروجات — ما خرج

```
expense_entries
├── id, business_id
├── amount, vat_amount, date
├── category       — materials / labor / tools / fuel /
│                    office / insurance / gov_fees / other
├── vendor_name
├── method         — cash / transfer / check
├── project_id     — (اختياري) ← projects
├── receipt_url    — صورة الإيصال / الفاتورة
├── note
└── created_at
```

---

### ❹ أرشيف الفواتير — للمحاسب فقط

> ⚠️ هذا أرشيف تخزين فقط — لا توليد فواتير إطلاقاً

```
invoice_archive
├── id, business_id
├── type           — received (واردة من مورّد)
│                    income_proof (إثبات دفع من عميل)
├── vendor_or_client_name
├── amount, vat_amount, date
├── category       — expense / income
├── project_id     — (اختياري)
├── client_id      — (اختياري)
├── file_url       — صورة / PDF الفاتورة
├── note
├── is_sent_to_accountant  — هل أُرسلت للمحاسب؟
└── created_at
```

---

### ❺ قسائم الرواتب

```
payroll_slips
├── id, business_id
├── worker_id      → workers
├── project_id     — (اختياري)
├── period_start, period_end
├── work_days, daily_rate
├── base_salary
├── deductions
├── advances_paid
├── net_pay
├── paid_at
├── pdf_url
└── created_at
```

---

### ❻ ملف المحاسب — الحزمة النهائية

```
accountant_packages
├── id, business_id
├── period_label   — "مايو 2026" / "Q2 2026" / "2026"
├── from_date, to_date
├── total_income
├── total_expenses
├── net_profit
├── total_vat_paid     — مجموع الضريبة على المشتريات
├── income_count       — عدد المدخولات
├── expense_count      — عدد المصاريف
├── archive_count      — عدد الفواتير المؤرشفة
├── notes_to_accountant
├── pdf_url
├── sent_at
└── created_at
```

## المستوى الثاني: هيكل الشاشات

```
FinanceScreen
│
├── 🏢 Business Switcher (شريط علوي ثابت)
│   ├── اسم المصلحة النشطة + رقمها
│   ├── Dropdown: قائمة مصالحك
│   └── + إضافة مصلحة جديدة
│
│   ──── كل شيء تحت = بيانات المصلحة المختارة ────
│
├── 📊 الرئيسية / Overview
│   ├── بطاقة: مدخولات الشهر
│   ├── بطاقة: مخروجات الشهر
│   ├── بطاقة: صافي الشهر
│   └── رسم بياني: آخر 6 أشهر
│
├── 💰 المدخولات
│   ├── قائمة + فلترة (شهر / مصدر / مشروع)
│   └── + تسجيل مدخول جديد
│       ├── المبلغ + التاريخ
│       ├── طريقة الدفع
│       ├── ربط بعميل (اختياري)
│       ├── ربط بمشروع (اختياري)
│       └── رفع صورة إثبات
│
├── 💸 المخروجات
│   ├── قائمة + فلترة (شهر / فئة / مشروع)
│   ├── رسم بياني بالفئات
│   └── + تسجيل مصروف جديد
│       ├── المبلغ + الضريبة + التاريخ
│       ├── الفئة + اسم المورّد
│       ├── ربط بمشروع (اختياري)
│       └── رفع صورة الإيصال
│
├── 🗂️ أرشيف الفواتير
│   ├── قائمة الفواتير المحفوظة
│   ├── فلترة: شهر / نوع / حالة الإرسال للمحاسب
│   ├── + رفع فاتورة جديدة (صورة/PDF)
│   └── تعليم "أُرسلت للمحاسب" ✓
│
├── 👷 قسائم الرواتب
│   ├── اختر العامل + الفترة
│   ├── حساب تلقائي من بيانات التطبيق
│   ├── معاينة القسيمة
│   └── توليد PDF + إرسال
│
└── 📁 ملف المحاسب
    ├── اختر الفترة
    ├── ملخص: مدخولات / مخروجات / صافي / ضريبة
    ├── قائمة بكل الوثائق للفترة
    └── تصدير PDF موحّد
```
