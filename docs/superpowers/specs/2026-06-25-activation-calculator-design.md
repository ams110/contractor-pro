- **التاريخ:** 2026-06-25

- **الصلة:** بند ٢ في [LAUNCH_PLAN.md](../../LAUNCH_PLAN.md) («كسر جرف التفعيل»)

## 1. المشكلة والهدف
- **صفر تفعيل**: ٥ من ٦ مسجّلين خارجيين لم يفعّلوا — السبب: splash إجباري 3.3ث → بوّابة إنشاء مصلحة إجبارية → **داشبورد فاضي** بلا إرشاد ولا قيمة فورية.
- **لحظة الأها** (مُختارة): **الراتب المحسوب تلقائياً** (ساعات إضافية 125%/150% + خصومات) — أكثر شي مميّز عن المنافسين وأقرب لوجع المقاول.
- **الهدف المزدوج**:
  - **اكتساب**: حاسبة راتب مجانية عامّة (بلا تسجيل) تقود الزائر بالتمييز قبل أي التزام → تسجيل.
  - **تفعيل**: نفس التجربة بعد التسجيل تحوّل المستخدم لأول **عامل حقيقي** بثواني → تقليل الوقت لأول قيمة.

## 2. السياق الحالي (الكود)
- التدفّق: `signup → /welcome (splash 3.3ث) → /app → FirstTimeSetup (بوّابة) → داشبورد فاضي`.
- [WelcomePage.jsx](../../../src/pages/WelcomePage.jsx): splash مزخرف بلا قيمة وظيفية (fade عند 2.7ث، تنقّل عند 3.3ث).
- [FirstTimeSetup.jsx](../../../src/screens/onboarding/FirstTimeSetup.jsx): زرّ واحد «عامة» (`handleAutoCreate`: ينشئ `osek_patur` + احتفال) أو `BusinessSetup` يدوي.
- `calcSalary(rate, dayType, hours)` في [helpers.js](../../../src/lib/helpers.js): يومي، الساعات 9-10 ×1.25، 11+ ×1.5. **يُعاد استخدامها كما هي** (لا منطق جديد).
- [Router.jsx](../../../src/Router.jsx): توجيه يدوي؛ المسارات العامّة (`/pricing`,`/terms`...) ملفوفة بـ`CookieConsent`.
- [seoRoutes.js](../../../src/lib/seoRoutes.js): مصدر SEO لكل مسار (prerender + sitemap تلقائيان postbuild).

## 3. المعمارية

### 3.1 `SalaryCalculator.jsx` (مكوّن مشترك — جديد)
- **المدخلات**: الأجر اليومي (₪) · ساعات اليوم · عدد الأيام. قيم افتراضية تُظهر الإضافي فوراً (مثلاً 400 / 10 / 22).
- **الحساب الحيّ**: `dayPay = calcSalary(dailyWage, <نوع اليوم العادي>, hoursPerDay)` ثم `monthTotal = dayPay × days` (تبسيط معاينة: نفس الساعات كل يوم). تفصيل: «8 ساعات عادية + س ساعة إضافية ×125%/150%».
- **props**: `mode: 'public' | 'onboarding'`، `onCta(values)`، `ctaLabel`.
- **العرض**: عدّاد متحرّك (`tabular-nums`)، تفصيل الإضافي، سطر تلميح «+ خصومات وמע"מ تُحسب داخل التطبيق» (غير تفاعلي في v1).
- **النقاء**: منطق الحساب يستدعي `calcSalary` فقط — لا منطق ضريبي مكرّر.

### 3.2 `CalculatorPage.jsx` (مسار عام `/calculator` — جديد)
- يلفّ `SalaryCalculator` بوضع `public` + هيدر/فوتر الهبوط + `useRouteSeo('/calculator')`.
- **CTA أساسي**: «سجّل مجاناً واحفظ عمّالك وأدِر رواتبهم» → `navigate('/register')`.
- **زر مشاركة واتساب**: نص ترويجي + رابط `/calculator` (حلقة فيروسية).

### 3.3 التفعيل (تعديل [App.jsx](../../../src/App.jsx))
- **شرط المستخدم الجديد**: لا `businesses` (نفس شرط `FirstTimeSetup` الحالي).
- نعرض تجربة onboarding مبنيّة على `SalaryCalculator` بوضع `onboarding`. الـ`onCta`:
  1. يطلب **اسم العامل** (حقل واحد).
  2. إذا لا مصلحة → ينشئ «عامة» (`osek_patur`) عبر `useBusinessStore.create` (نفس منطق `handleAutoCreate`).
  3. ينشئ العامل عبر هوك إنشاء العمّال الموجود (`useEmployees` من `useData.js`) بالأجر اليومي المُدخل.
  4. احتفال + دخول التطبيق.
- نُبقي خيار **«إعداد يدوي»** → `BusinessSetup` (المسار القديم سليم).
- **تلميح خفيف** بعد الدخول: flag في `localStorage` يُظهر بطاقة «سجّل أول يوم عمل تشوف الراتب يتراكم» → تفتح إضافة يوم عمل؛ تختفي بعد أول `work_day` أو عند الإغلاق.

### 3.4 إزالة الـsplash الإجباري
- مسار ما بعد الدخول: لا توجيه إجباري لـ`/welcome`. **جديد → onboarding**، **عائد → `/app`** مباشرة.
- `WelcomePage` تبقى كملف لكن تُرفَع من المسار الحرج (تُزال نقطة استدعاء `navigate('/welcome')` — على الأرجح في `LoginScreen`؛ تُحدَّد بدقّة في خطة التنفيذ).

## 4. تدفّق البيانات
- **عام**: مدخلات → `calcSalary` → عرض (لا تخزين). CTA → `/register`.
- **تفعيل**: مدخلات → `calcSalary` → عرض. CTA → `useBusinessStore.create` (+ هوك إنشاء العامل) → تحديث `useDataStore`/Realtime → دخول.

## 5. معالجة الأخطاء
- مدخلات سالبة/فارغة → النتيجة 0 + تحقّق ناعم، بلا crash.
- فشل إنشاء المصلحة/العامل → رسالة خطأ وإبقاء المستخدم بالشاشة (نمط `handleAutoCreate` الحالي).
- الحاسبة العامّة **حساب محلي بالكامل** → تعمل بلا إنترنت.

## 6. SEO (الاكتساب)
- إضافة `/calculator` لـ`ROUTE_SEO` (عنوان موجّه: «احسب راتب عامل بالساعات الإضافية مجاناً | كبلان» + وصف + FAQ اختياري) → يدخل prerender + sitemap تلقائياً (لا `noindex`). يستهدف بحث «حساب راتب عامل ساعات إضافية إسرائيل» بالعربي.
- رابط/قسم من [LandingPage.jsx](../../../src/pages/LandingPage.jsx) → الحاسبة.

## 7. الاختبار
- **وحدة**: منطق الحاسبة المعروض يطابق `calcSalary` بالضبط — حالات ≤8ث، 9-10ث (×1.25)، 11+ث (×1.5)، و«كامل/نص يوم/عطلة».
- **(اختياري) E2E**: `/calculator` يعرض ويحسب؛ تسجيل → onboarding → التحقّق أن عاملاً أُنشئ.

## 8. النطاق و YAGNI
- **v1**: أجر + ساعات + أيام + تفصيل الإضافي (القلب). اختيار **نوع العامل/الخصومات التفاعلية** = fast-follow بعد v1.
- ❌ لا بيانات ديمو مزروعة · ❌ لا مكتبة coachmarks · ❌ لا لمس محرّك الرواتب (إعادة استخدام فقط) · ❌ لا auth على الحاسبة العامّة · ❌ لا إنشاء يوم عمل تلقائي عند الحفظ (بدّو مشروع — يبقى خطوة الإرشاد).
